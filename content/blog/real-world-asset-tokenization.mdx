The biggest narrative in DeFi isn't another yield farm—it's **real-world asset (RWA) tokenization**. BlackRock, Franklin Templeton, and JP Morgan are now tokenizing billions in traditional assets. This isn't coming. It's here.

## What Are RWAs?

**Real-world assets** = traditional financial assets represented as blockchain tokens.

**Categories:**
1. **US Treasuries** (BlackRock BUIDL: $1.5B+)
2. **Private credit** (Centrifuge, Goldfinch)
3. **Real estate** (RealT, Propy)
4. **Commodities** (Paxos Gold)
5. **Invoices/receivables** (Maple Finance)

**Why tokenize?**
- 24/7 trading (no market hours)
- Fractional ownership ($100 → slice of $10M bond)
- Programmable yield (auto-compounding)
- Global access (anyone with wallet)
- Transparent pricing (on-chain oracles)

## Case Study: BlackRock BUIDL

**Launch:** March 2024  
**AUM (Feb 2026):** $1.5B+  
**Yield:** ~5.2% APY

**Structure:**
```
Investor → Buys BUIDL token ($1/token)
    ↓
BlackRock → Invests in US T-bills + repo agreements
    ↓
Yield → Distributed as new BUIDL tokens (auto-compounding)
    ↓
Redemption → Burn BUIDL, receive USD (T+1 settlement)
```

**Key features:**
- **Instant settlement:** T-bills settle in 24 hours (vs. 2-3 days traditional)
- **No minimums:** $1 minimum (vs. $1M for institutional T-bills)
- **Composable:** Can use BUIDL as DeFi collateral
- **Regulatory:** SEC-registered, qualified purchaser only

**Legal wrapper:**
- BUIDL = security token (Reg D offering)
- Issued via BNY Mellon (custodian)
- KYC/AML via Securitize (issuance platform)

## Tokenization Infrastructure

### 1. Legal Structure

**Challenge:** Link blockchain token to off-chain asset legally.

**Solution: Special Purpose Vehicle (SPV)**

```
┌──────────────────────────────────────┐
│  Investor (buys token)               │
└────────────┬─────────────────────────┘
             │ $USD
             ▼
┌──────────────────────────────────────┐
│  SPV (Cayman/Delaware)               │
│  - Holds off-chain assets            │
│  - Issues tokens (1:1 with assets)   │
└────────────┬─────────────────────────┘
             │ Assets
             ▼
┌──────────────────────────────────────┐
│  Custodian (BNY Mellon, State Street)│
│  - Holds US Treasuries               │
│  - Reports holdings daily            │
└──────────────────────────────────────┘
```

**Token smart contract:**
```solidity
// Simplified RWA token
contract RWAToken is ERC20 {
    address public custodian; // Off-chain asset holder
    string public assetReportURI; // IPFS/HTTP link to holdings
    
    mapping(address => bool) public whitelisted; // KYC'd investors
    
    function transfer(address to, uint256 amount) 
        public 
        override 
        returns (bool) 
    {
        require(whitelisted[msg.sender], "Not KYC'd");
        require(whitelisted[to], "Recipient not KYC'd");
        return super.transfer(to, amount);
    }
    
    // Admin functions (only custodian)
    function mint(address to, uint256 amount) external {
        require(msg.sender == custodian, "Unauthorized");
        _mint(to, amount);
    }
    
    function burn(address from, uint256 amount) external {
        require(msg.sender == custodian, "Unauthorized");
        _burn(from, amount);
    }
}
```

**Key insight:** Token is legally a claim on SPV's assets. If SPV goes bankrupt, token holders have first claim (not unsecured creditors).

## Yield Generation Mechanisms

### 1. Pass-Through Yield (BlackRock BUIDL Model)

**How it works:**
- SPV earns yield from US T-bills (~5.2%)
- Daily distribution as new tokens
- Token holder's balance auto-increases

**Example:**
```
Day 0: Hold 1,000 BUIDL ($1,000)
Day 30: Hold 1,004.5 BUIDL ($1,004.50)
         ↑ +$4.50 yield (0.45% monthly = 5.4% APY)
```

**Tax treatment (US):**
- Taxed as interest income (ordinary income rates)
- Reportable annually (1099-INT or similar)

---

### 2. Staking Rewards (Centrifuge Model)

**How it works:**
- Invest in tokenized private credit pool
- Earn yield from borrower interest payments
- Claim rewards manually

**Example (Centrifuge pool):**
```javascript
// Invest in private credit pool
await centrifuge.invest(POOL_ID, 100000e6); // $100k USDC

// After 30 days, claim yield
const earned = await centrifuge.claimRewards(POOL_ID);
// Returns: $800 (8% APY on $100k)
```

**Risk:** Credit default (borrower doesn't repay)
**Mitigation:** Pools diversified across 20-50 loans

---

### 3. Collateral-Based Yield (MakerDAO Model)

**How it works:**
- Use RWA tokens as collateral in DeFi
- Borrow stablecoins against them
- Earn spread (RWA yield - borrow cost)

**Example:**
```
1. Hold $1M BUIDL (earning 5.2%)
2. Deposit as collateral in Aave
3. Borrow $800k USDC (80% LTV) at 3.5%
4. Re-invest $800k USDC in more BUIDL

Net yield: 
  - Earn 5.2% on $1.8M = $93,600
  - Pay 3.5% on $800k = -$28,000
  - Net: $65,600 = 6.56% on $1M (leverage boost)
```

**Risk:** Liquidation if BUIDL price drops (rare for T-bills, but possible in crisis)

## Regulatory Landscape

### US: SEC Guidance (2023-2026)

**Key rulings:**
1. **RWA tokens = securities** (subject to SEC registration)
2. **Exemptions:** Reg D (accredited investors), Reg S (non-US)
3. **Custody:** Must use qualified custodian (Coinbase, Anchorage)
4. **Reporting:** Form D, annual audits

**Compliance checklist:**
```markdown
✅ Accredited investor verification
✅ Transfer restrictions (whitelisting)
✅ KYC/AML (FinCEN requirements)
✅ Form D filing (within 15 days of first sale)
✅ Annual audit (PCAOB-registered firm)
✅ Disclosure documents (offering memorandum)
```

---

### EU: MiCA (2024 Implementation)

**RWA treatment under MiCA:**
- **Asset-referenced tokens (ARTs):** Backed by basket of assets
- **E-money tokens (EMTs):** Backed by fiat currency
- **Licensing:** CASP (Crypto-Asset Service Provider) required

**Impact:**
- Stablecoin issuers need e-money license
- RWA token issuers may need ART license
- Passporting: License in one EU country = valid across EU

---

### Emerging: Tokenization-Specific Regulation

**UK Financial Services and Markets Bill (2026):**
- Creates "digital securities" category
- Sandbox for tokenized assets
- FCA oversight

**Singapore MAS (2025):**
- DPT (Digital Payment Token) framework
- STO (Security Token Offering) guidelines
- Simplified process for tokenized bonds

## Risk Assessment

### 1. Credit Risk

**For private credit RWAs:**
- Borrower default
- Under-collateralization

**Mitigation:**
- Diversify across 20+ pools
- Invest in senior tranches (first loss protection)
- Monitor credit ratings (S&P, Moody's for rated pools)

---

### 2. Smart Contract Risk

**For all RWAs:**
- Contract exploit
- Oracle manipulation
- Admin key compromise

**Mitigation:**
- Use audited platforms (Securitize, Centrifuge)
- Verify oracles (Chainlink preferred)
- Multi-sig admin controls

---

### 3. Legal Risk

**For cross-border RWAs:**
- Jurisdiction disputes
- Regulatory changes
- SPV bankruptcy

**Mitigation:**
- Legal opinion from crypto-specialized firm
- Jurisdiction selection (Delaware, Cayman for SPVs)
- Bankruptcy-remote structure
